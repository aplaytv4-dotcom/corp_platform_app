ЧТО УЛУЧШИТЬ В ПРОЕКТЕ
======================

1. Улучшить Django admin
- добавить более удобные list_display
- добавить фильтры
- добавить поиск
- добавить list_select_related
- добавить autocomplete_fields
- сделать inline-редактирование там, где это уместно
- привести названия разделов и полей к более понятному виду

2. Улучшить администрирование пользователей
- сделать более удобную форму создания и редактирования пользователей
- продумать валидацию role / scope_type / management / department в admin form
- скрывать лишние поля в зависимости от типа доступа

3. Улучшить экран сотрудников
- сделать корректные фильтры по отделу и активности
- добавить полноценный поиск
- вывести больше полезных текущих данных по назначению

4. Улучшить экран штатных единиц
- явно показывать статус занята / вакантна
- показывать текущего сотрудника более наглядно
- добавить фильтры по отделу, активности и занятости

5. Улучшить экран назначений
- сделать отдельные удобные формы:
  - создать назначение
  - закрыть назначение
  - перевод
- продумать UX закрытия назначения
- добавить больше защит от ошибочного закрытия и перевода

6. Улучшить frontend-валидацию
- унифицировать сообщения об ошибках
- сделать более аккуратную обработку ошибок API
- показать полевые ошибки backend прямо в формах

7. Улучшить справочники
- добавить больше контроля уникальности на UI
- сделать более удобное редактирование причин отсутствия, должностей, отделов, управлений

8. Улучшить отчеты
- сделать более аккуратный HTML preview
- улучшить внешний вид печати
- проверить формат Word/PDF на реальных данных

9. Улучшить безопасность и эксплуатацию
- разделить настройки на dev/prod
- добавить CORS при необходимости
- добавить logging
- добавить audit trail изменений

10. Улучшить тестирование
- добавить unit tests для services
- добавить API tests
- добавить smoke/e2e проверки ключевых сценариев

11. Улучшить документацию
- сделать отдельный README.md в нормальном репозиторном формате
- добавить краткую схему ролей и доступа
- добавить описание API сценариев

12. Улучшить git-процесс
- запушить проект в GitHub
- настроить осмысленные коммиты
- при необходимости добавить CI
